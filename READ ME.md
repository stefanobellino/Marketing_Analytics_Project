# Sales Analytics & Marketing A/B Testing Workflow

## Project Overview
This repository contains a comprehensive analytics project designed to simulate and evaluate a marketing A/B testing campaign (*Control* vs *Treatment* groups) across 10,000 customers and three consumer tiers. Real-world data is rarely pristine, so this project specifically addresses messy data workflows, tracking gaps, business intelligence, and autonomous AI analysis.

The workflow integrates **Python**, **SQL/MySQL**, **Business Intelligence (Power BI)**, and **Agentic AI** to analyze performance, handle missing values (NaNs), and generate strategic business insights.

---

## Dataset Source
The dataset used for this project is the **Sales Analytics with NaNs and Marketing A/B Test** dataset, originally published on Kaggle.
- **Dataset Link:** [Kaggle - Sales Analytics & Marketing A/B Test](https://www.kaggle.com/datasets/mjawad17/sales-analytics-nans-and-marketing-ab-test?resource=download)

### Dataset Description (from source):
> *In modern e-commerce and retail analytics, understanding the true impact of a marketing campaign requires robust experimental setups like A/B testing. However, real-world data is rarely pristine. Telemetry drops, skipped survey questions, and platform migration issues leave behind tracking gaps.*
> 
> *This dataset simulates a multi-segment marketing treatment campaign across 10,000 customers. It features comprehensive performance metrics across two distinct experimental tracks (Control vs. Treatment) and three consumer tiers. Crucially, realistic missingness (NaNs) has been injected into several columns to replicate the messy workflows that data scientists face daily.*
> 
> *Relationships between customer engagement, historical purchasing value, post-intervention satisfaction, and financial return (ROI) have been intentionally baked into the underlying distribution using Missing Completely at Random (MCAR) and Missing at Random (MAR) mechanisms.*

---

## Tools & Technologies Used
- **MySQL:** Relational database management system used for data storage, cleaning, and querying aggregated campaign metrics.
- **Python (Pandas, Connector):** Used for data cleaning/imputation workflows, database interaction, and running agentic automation.
- **Agentic AI (Google Gemini API - `gemini-2.5-flash`):** An autonomous AI data analyst agent that inspects database results, identifies anomalies (such as satisfaction drops in the *Medium Value* segment), and drafts strategic executive summaries for the CMO.
- **Power BI:** Business Intelligence tool used to build interactive visualizations and dashboards tracking incremental revenue, ROI validation, and experimental tracks.
